## Context

这是一个面向 3-6 岁幼儿的纯前端英语启蒙网页，作为仓库里"椰子"系列的新成员。目标用户不识字，交互必须以图标、声音、动画为主。仓库现有项目（如 MathKidsAdventure、MathPlayground）均为零依赖的静态 HTML/CSS/JS，本项目沿用同样的技术约束，无需后端、无构建步骤。

## Goals / Non-Goals

**Goals:**
- 打开 `index.html` 即可运行，无依赖、无构建。
- 三大模块：主题单词卡、字母 ABC、听音找图游戏，复用同一套发音与声音开关机制。
- 大按钮、明亮护眼配色、触屏友好，单文件单页应用（屏幕切换由 JS 控制显隐）。
- 内容数据与逻辑分离，便于后续增删主题和单词。

**Non-Goals:**
- 不做用户账号、进度持久化（首版不接 localStorage 排行）。
- 不做录音、语音识别等需要麦克风权限的功能。
- 不引入音频素材文件，发音完全依赖 Web Speech API。
- 不做多语言界面切换（界面中文，学习内容英文）。

## Decisions

**1. 单页 + 屏幕切换，而非多 HTML 页面**
所有模块放在一个 `index.html` 内，用 `.screen` 区块的显隐切换。理由：状态（声音开关、得分）在内存中共享简单，无页面跳转闪烁。备选方案多页面会让发音状态和声音开关难以共享。

**2. 发音用 Web Speech API（speechSynthesis），不打包音频**
零体积、覆盖任意单词、易扩展。备选：预录 mp3，体积大且每加一个词都要录音，弃用。封装一个 `speak(text)` 函数，统一处理：声音开关判断、`cancel()` 防排队、英文嗓音选择、慢语速（rate≈0.85）。

**3. 数据驱动（data.js）**
主题、单词、字母、代表词都放在 `data.js` 的常量对象里，`app.js` 只负责渲染与交互。新增主题或单词只改数据，不动逻辑。

**4. 图标采用内联 SVG 卡通插画，而非 emoji**
为达到"可爱卡通、高清、非 emoji"的诉求，图标用手绘内联 SVG（圆润描边、明亮配色的可爱风），矢量天然高清、任意缩放不糊，且零外部文件、零依赖。备选：(a) emoji——风格受系统控制、偏符号化，已弃用为主图标；(b) 位图插画素材包/AI 生成——质感更精致，但需引入文件或外部生成工具。
为兼顾后续升级，渲染层通过统一的图标查找函数（如 `getIcon(key)` 返回 SVG 字符串）取图，数据只存图标 key。日后若要换成位图素材或 AI 生成插画，只改 `getIcon` 的实现与素材，不动业务逻辑与数据。

**5. 听音找图的干扰项生成**
从同主题（或全词池）里随机抽取 N-1 个非目标词作为干扰项，与目标项混合后打乱。同主题干扰更有教学意义（如水果间区分）。

**6. 休息提醒计时器**
用 `setInterval`/`setTimeout` 累计前台使用时长，每满 20 分钟弹出一个覆盖层提示休息。计时只在页面可见时累计（监听 `visibilitychange`，切走暂停），弹框点击"好的"后重置并继续计时。弹框出现时暂停语音。备选累计真实日历时间不准确（孩子可能离开），故用前台活跃时长。

## Risks / Trade-offs

- [Web Speech API 兼容性/嗓音差异] → 启动时探测，无可用语音则静默降级；优先 en 系嗓音，找不到则用默认。
- [移动端首次发音需用户手势触发] → 所有发音都由点击触发，天然满足浏览器自动播放策略。
- [emoji 在不同系统渲染不一致] → 主图标改用内联 SVG 卡通插画，不再依赖系统 emoji 渲染；仅在极少数装饰处可保留 emoji。
- [手绘 SVG 覆盖大量单词的工作量与精致度有限] → 首版聚焦"可爱简笔卡通"覆盖核心词表；通过 `getIcon` 抽象预留升级到位图/AI 插画的能力，不返工逻辑。
- [语音排队堆叠导致卡顿] → 每次 `speak` 前调用 `speechSynthesis.cancel()`。
- [无进度保存] → 首版可接受；如需要后续用 localStorage 增量添加，不影响现有结构。
