# 数学小乐园 - 需求文档

## 项目概述

开发一款面向幼儿园至一年级前（3-6岁）小朋友的数学启蒙网页互动游戏。UI风格可爱有趣，覆盖学前基础数学能力，支持多角色切换和难度自定义。

## 目标用户

- 主要用户：3-6岁幼儿（需家长辅助操作）
- 次要用户：家长（配置难度、查看进度）

## 核心功能模块

### 1. 数数游戏
- 画面出现若干可爱物品（水果、动物等），小朋友点击计数后选择正确数字
- 支持正数和倒数
- 难度随数字范围提升（10/20/100以内）

### 2. 加减法计算
- 基础加法和减法运算
- 可视化辅助（用图形表示数量帮助理解）
- 混合运算（进阶难度）
- 数字范围可配置
- 自由练习模式下可选择只练加法、只练减法或加减混合

### 3. 比大小
- 两组物品或两个数字，选出大的/小的
- 引入大于号、小于号、等于号的认识
- 难度：从直观图形比较 → 纯数字比较

### 4. 认识形状
- 基本形状识别：圆形、三角形、正方形、长方形、五角星
- 在生活场景中找形状
- 形状分类和配对

### 5. 找规律
- 颜色规律（红蓝红蓝→？）
- 形状规律（圆方圆方→？）
- 数字规律（2,4,6,?）
- 难度递增：从简单ABAB到更复杂模式

### 6. 排序
- 从小到大/从大到小排列数字
- 物品按大小排序（拖拽交互）
- 难度随数字范围和数量提升

## 游戏系统

### 模式设计
- **闯关模式**：分关卡逐步解锁，每关有明确目标
- **自由练习模式**：选择任意模块自由练习，不限次数
- **冒险故事模式**：帮助小动物角色解决生活中的数学问题（情境化）

### 角色系统
- 默认主角：椰子🥥
- 可解锁/切换其他角色（如小猫🐱、小兔🐰、小熊🐻）
- 切换角色不重置学习进度
- 角色作为陪伴引导，给予鼓励反馈

### 难度系统
- 用户可选择数字范围：10以内（默认）/ 20以内 / 100以内
- 修改难度后所有模块统一调整
- 难度影响：数字大小、题目复杂度、选项干扰项数量

### 进度与奖励
- 星星积分系统（答对得星星）
- 关卡解锁进度保存（localStorage）
- 成就勋章系统（完成特定目标解锁）
- 进度跨角色共享

## UI/UX 要求

### 视觉风格
- 可爱、明亮、卡通风格
- 大按钮、大字体，适合幼儿点击
- 丰富的 emoji 和 CSS 动画
- 柔和的配色方案（粉色、天蓝、嫩绿为主色调）

### 交互设计
- 支持点击和拖拽两种交互
- 即时反馈（答对/答错动画和音效）
- 语音播报题目（Web Speech API）
- 触屏友好，适配平板和手机

### 响应式
- 优先适配平板横屏
- 兼容手机竖屏
- PC浏览器也可正常使用

## 技术约束

- 纯前端实现：HTML5 + CSS3 + JavaScript
- 无需后端服务器
- 数据存储使用 localStorage
- 不依赖第三方框架（Vanilla JS）
- 音效使用 Web Audio API
- 语音使用 Web Speech Synthesis API

## 护眼与健康

### 使用时长提醒
- 连续使用20分钟后弹出休息提示框
- 提示框风格可爱友好（如角色说"眼睛累了，休息一下吧！"）
- 提示框出现时暂停游戏
- 用户点击"休息好了"按钮后继续游戏，计时器重新开始
- 计时从页面打开/上次确认休息后开始计算

## 非功能需求

- 页面加载时间 < 2秒
- 动画流畅不卡顿（60fps）
- 无广告、无外部链接（儿童安全）
- 所有文字使用中文
